## 🔄 Les 4 Phases Shape Up

### 1. **SHAPING** (Semaine -2 à 0)
**Objectif** : Définir les limites avant de s'engager

- Identifier le problème utilisateur
- Définir l'appetite (FIXE)
- Breadboarding & Fat Marker Sketching
- Identifier les rabbit holes
- Écrire le pitch

**Output** : Pitch validé pour betting table

### 2. **BETTING TABLE** (Semaine 0)
**Objectif** : Priorisation brutale et engagement

- Présenter le pitch (15 min)
- Décision finale (vote, engagement)

**Output** : Projet validé pour cycle

### 3. **BUILDING** (Semaine 1-3)
**Objectif** : Livrer une feature shippable

- Semaine 1-2 : Get One Piece Done
- Semaine 3 : Map Scopes
- Jour 15 : Downhill Execution

**Output** : Feature shippable

### 4. **COOLDOWN** (Semaine 4)
**Objectif** : Repos, amélioration, innovation

- Bug fixes prioritaires
- Technical debt paydown
- Exploration technique
- Documentation

**Output** : Projet stabilisé, prêt pour prochain cycle
